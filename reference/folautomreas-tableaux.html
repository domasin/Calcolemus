unify_literals env tmp

Full Usage: unify_literals env tmp

Parameters:
    env : func<string, term> - An accumulator of the environment of the variable assignments, maintained globally and represented as a cycle-free finite partial function.
    tmp : formula<fol> * formula<fol> - The pair of litterals to be unified. It also handles the pair False,False.

Returns: func<string, term> A variable-term mappings that unify tmp, if the unification is possible and there is no cycle. If the input pair is already unified or the input mappings already unify it, unify_literals just returns the input mappings, otherwise it augments the input mappings with the term-variable assignments needed for the unification.

Unifies an input pair of litterals.

env : func<string, term>

An accumulator of the environment of the variable assignments, maintained globally and represented as a cycle-free finite partial function.

tmp : formula<fol> * formula<fol>

The pair of litterals to be unified. It also handles the pair False,False.

Returns: func<string, term>

A variable-term mappings that unify tmp, if the unification is possible and there is no cycle. If the input pair is already unified or the input mappings already unify it, unify_literals just returns the input mappings, otherwise it augments the input mappings with the term-variable assignments needed for the unification.

Exception
  • with message 'cyclic' when there is a cyclic assignment;
  • with message 'impossible unification' when the unification is not possible.
  • with message 'Can't unify literals' when the input tmp is neither a pair of literal or the degenerated case False,False.
Example

An example of successful unification: \(x \mapsto f(y)\) 

 unify_literals undefined (!!"P(x)",!!"P(f(y))")
An example of cyclic assignment: 
 unify_literals undefined (!!"P(y)",!!"P(f(y))")
An example of impossible unification: 
 unify_literals undefined (!!"P(g(x))",!!"P(f(y))")
