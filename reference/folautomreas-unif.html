solve env

Full Usage: solve env

Parameters:
    env : func<string, term> - The environment of mappings to be reduced.

Returns: func<string, term>
  • a new environment of variable-term mappings that is an MGU of env, if there are useless mappings to be removed;
  • the same input env, if it is already an MGU.

Removes useless mappings from an environment of variable-term mappings env returned by Unif.unify, giving a most general unifier (MGU).

The function never fails. However, it would cause a StackOverflow crash if called on a cyclic env. Unif.unify is specifically designed not to produce cyclic mappings.

env : func<string, term>

The environment of mappings to be reduced.

Returns: func<string, term>

  • a new environment of variable-term mappings that is an MGU of env, if there are useless mappings to be removed;
  • the same input env, if it is already an MGU.

Example

The following reduces \(x \mapsto y, x \mapsto 0\) to \(x \mapsto 0\). 

 solve (("x" |-> Fn("0",[]))(("x" |-> Var "y")undefined))
The following returns the input unchanged \(x \mapsto y, y \mapsto 0\) 
 solve (("y" |-> Fn("0",[]))(("x" |-> Var "y")undefined))
The following would cause a StackOverflow crash. 
 solve (("y" |-> Fn("f",[Var "y"]))undefined)

unify env eqs

Full Usage: unify env eqs

Parameters:
    env : func<string, term> - An environment of mappings (represented as a finite partial function) from variables to terms, used as an accumulator for the final result of the unification procedure.
    eqs : (term * term) list - The list of term-term pairs to be unified.

Returns: func<string, term> A variable-term mappings that unify eqs, if the unification is possible and there is no cycle.

Main unification procedure.

It applies some transformations to eqs and incorporates the resulting variable-term mappings into env. env might contain mappings that could map a variable to a term containing other variables that are themselves assigned: for example \(x \mapsto y\) and \(y \mapsto z\) instead of just \(x \mapsto z\) directly. The call to Unif.istriv guarantees that there is no cycle or detects it and stops immediately the unification process with a failure.

env : func<string, term>

An environment of mappings (represented as a finite partial function) from variables to terms, used as an accumulator for the final result of the unification procedure.

eqs : (term * term) list

The list of term-term pairs to be unified.

Returns: func<string, term>

A variable-term mappings that unify eqs, if the unification is possible and there is no cycle.

Exception
  • with message 'cyclic' when there is a cyclic assignment;
  • with message 'impossible unification' when the unification is not possible.
Example

Success with no previous assignment: \(x \mapsto 0\) 

 unify undefined [Var "x", Fn("0",[])]
Success with augmented assignment \(x \mapsto y, y \mapsto 0\) 
 unify (("x" |-> (Var "y"))undefined) [Var "x", Fn("0",[])]
Failure with direct cycle: \(y \mapsto f(y)\) 
 unify undefined [Var "y", Fn("f",[Var "y"])]
Failure with derived cycle: \(x \mapsto y, y \mapsto f(y)\) 
 unify (("x" |-> (Var "y"))undefined) [Var "x", Fn("f",[Var "y"])]
Impossible unification: 
 unify undefined [Fn ("0",[]), Fn("1",[])]
